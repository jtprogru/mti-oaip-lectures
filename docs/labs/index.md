## Итоговая практика

Большой сквозной проект «распределённая система оплаты услуг» — см. [практическое задание](../practice/practice.md). Объединяет ООП, HTTP, БД, GUI, логирование и тестирование; зачитывается как итоговая лабораторная.

## Как сдавать

1. Каждая лаба — отдельный коммит (или PR) в репозиторий, который вы создадите по образцу из лекции про [системы контроля версий](../practice/version-control.md).
2. Структура: исходный код, `README.md` с описанием задания и инструкцией запуска, тесты (где применимо).
3. Все Python-проекты — на `uv` (см. [тему 5](../topics/05-libraries-and-modules/lecture-01-modules-packages.md)); Go-проекты — на `go mod` (см. [тему 13](../topics/13-golang-basics/lecture-01-intro.md)).
4. Стиль: `ruff format`/`ruff check` для Python, `gofmt`/`golangci-lint` для Go — должны проходить чисто.
